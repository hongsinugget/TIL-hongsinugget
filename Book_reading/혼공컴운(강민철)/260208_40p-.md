## 컴퓨터 4가지 핵심 부품 (40p~)

**메모리**

> 메모리는 현재 실행되는 프로그램의 명령어와 데이터를 저장하는 부품

컴퓨터가 빠르게 작동하려면 저장된 명령어와 데이터들이 위치가 정돈되어 있어야 한다.

그래서 메모리에는 빠르게 효율적으로 접근하려면 **주소**를 사용한다.

**CPU**

cpu는 메모리에 저장된 명령어를 읽고 해석하고 실행하는 부품이다.

CPU 내부 구성요소 3가지

- 산술논리연산장치(ALU)
- 레지스터
- 제어장치

**ALU**는 계산을 위한 부품

**레지스터**는 임시기억장치

**제어장치**는 제어신호로 컴퓨터 부품을 관리하고 작동시키는 일종의 전기 신호

메모리를 저장된 값을 읽고싶을땐 **메모리 읽기** 제어신호를 보낸다

어떤 값을 저장하고싶을땐 **메모리쓰기** 제어신호를 보낸다

<순서>

먼저 저장된 명령어를 읽기 위해 메모리에 메모리 읽기 신호를 보냄 -> 메모리는 저장된 명령어를 CPU에 보냄->레지스터에 저장->제어 장치는 읽은 명령어를 해석하고 그 다음 저장된 데이터가 필요하다 판단-> 제어장치는 또 다음 저장된 데이터를 읽기위해 메모리 읽기 신호를 보낸다.

<두번째 순서>

다음 저장된 데이터를 CPU에 전달 -> 각각 다른 레지스터에 저장->ALU는 읽은 데이터를 연산을 수행 ->계산 결과값을 레지스터에 저장 (계산이 끝나면 명령어 실행 끝)

**보조기억장치**

기존 메모리보다 크기가 크고 전원이 꺼져도 저장을 잃지않는 보조기억장치가 필요하다

> 하드디스크,SSD,USB등 저장장치가 보조기억장치

**입출력장치**

> 마우스 키보드처럼 컴퓨터 외부에서 연결되어 내부의 정보를 교환하는 방식

**메인보드와 시스템버스**

컴퓨터 핵심부품들은 메인보드 판에 연결된다.

메인보드에 연결된 부품들은 다양한 종류의 통로,버스가 있다
그중 컴퓨터 네가지 핵심 부품을 연결하는 중요한 버스는 **시스템버스**다

시스템 버스에는 주소를 주고받는 통로인 **주소버스**

명령어와 데이터를 주고받는 통로 **데이터 버스**

제어신호를 주고받는
**제어버스**

> 여기서 메모리를 읽을때 제어신호만 보내는게 아니라 제어버스로 메모리읽기 제어신호를 보내고 주소버스로 읽고자 하는 주소를 보낸다.
그리고 메모리는 데이터 버스로 CPU가 요청한 주소에 있는 내용을 보낸다.
